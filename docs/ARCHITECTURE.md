The Sitelayer system is a distributed application composed of a web frontend, an API backend, a background worker, and several shared domain packages. At its core, Sitelayer provides construction takeoff and estimation tools, enabling users to manage projects, calculate geometry for blueprints, and synchronize data with external integrations such as QuickBooks Online.

The web frontend is a React application built with Vite, utilizing React Router for navigation and TailwindCSS for styling. It implements an offline-first architecture by maintaining an offline mutation queue. When network connectivity is lost, user actions are queued locally. Once the connection is restored, the application replays these offline mutations to the backend. The frontend communicates with the backend via REST API endpoints to fetch bootstrap data, project summaries, and takeoff measurements.

The API backend is a Node.js HTTP server backed by a PostgreSQL database. It handles incoming requests, manages authentication and rate limiting, and serves as the primary data ingestion point. When mutations occur, the API often writes to an outbox table (`mutation_outbox`) or sync events table rather than executing long-running operations inline. This allows the API to remain responsive while deferring integration tasks. The backend also interacts with cloud storage for managing blueprint documents and processing uploaded files.

The background worker is a standalone Node.js process responsible for consuming the mutation outbox and sync events. It periodically polls the database using a heartbeat mechanism. When pending events are detected, the worker leverages the shared queue package to process batches using row-level database locking. This ensures that integration events, such as syncing data to QuickBooks Online, are processed reliably and without duplication.

Shared business logic is encapsulated within the domain package. This package contains pure functions for calculating polygon areas, project costs, margins, and bonus payouts. By centralizing this logic, both the web frontend and the backend API can reliably perform the same calculations. Additional shared packages provide structured, context-aware logging and configuration management across all services. The entire system is instrumented with distributed tracing to provide visibility across frontend actions, API requests, and background worker processing.
