Sitelayer is a distributed construction takeoff and estimation system composed of a React frontend, Node.js backend, background worker, and shared libraries. The system supports multi-tier deployment (local, dev, preview, prod) with tier-aware configuration, feature flags, and storage backends.

The web frontend is a React application built with Vite and React Router, styled with TailwindCSS. It implements an offline-first architecture using a client-side mutation queue stored in localStorage. When offline, user actions are queued locally; upon reconnection, the application replays mutations to the backend. Failed mutations with conflict responses (409) are retained for manual resolution. The frontend fetches bootstrap data, project summaries, and measurements via REST API, with response caching to improve perceived offline availability.

The API backend is a Node.js HTTP server backed by PostgreSQL, handling request routing, authentication, and rate limiting. Rather than executing long-running tasks synchronously, mutations are written to an outbox table (`mutation_outbox`) or sync events table to keep the API responsive. Blueprint documents and uploaded files are managed through a pluggable storage adapter that supports both local filesystem (for local/dev tiers) and S3 (for preview/prod tiers). The tier configuration system determines database connections, feature flags, and storage backends at startup.

The background worker is a Node.js process that consumes the mutation outbox and sync events by polling the database at configurable intervals. The worker uses the shared queue package to acquire row-level database locks before processing batches, ensuring reliable processing without duplication. This pattern enables deferred execution of integration events such as syncing project data to QuickBooks Online.

Shared packages encapsulate cross-service concerns: the domain package contains pure functions for geometry calculations, project costing, margins, and bonus payouts; the config package manages tier-aware configuration and feature flags; the logger package provides structured, context-aware logging; and the queue package implements leased batch processing with database row locking. Distributed tracing via Sentry instruments all tiers, providing visibility across frontend mutations, API requests, and background processing.
