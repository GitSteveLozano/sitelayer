Sitelayer is a multi-tenant construction takeoff and estimation system built as a monorepo containing a React single-page frontend, a plain Node.js HTTP API, a background worker, and shared TypeScript packages. The system enforces strict tier-aware configuration across local, dev, preview, and prod environments, verifying database and storage bindings to prevent cross-environment contamination.

The web frontend operates as a resilient, offline-capable React application using React Router and TailwindCSS alongside Radix UI for component styling. Upon loading, it fetches a comprehensive bootstrap payload of reference data, lazily loading project-specific entities like blueprints and schedules. To handle network disruptions, the frontend queues mutations locally and runs a background synchronization loop that replays them to the backend when connectivity is restored.

The Node.js API server processes incoming requests directly using the native HTTP module without a heavy web framework. It manages routing, JSON parsing, and CORS while enforcing multi-tenancy through explicit request headers rather than complex sessions. The API interacts with a PostgreSQL database, executing synchronous reads and writes but deferring integration-heavy tasks—such as QuickBooks Online synchronization—to mutation outbox and sync events tables. It also records audit logs for entity changes. Blueprint storage is abstracted behind an adapter supporting the local filesystem or DigitalOcean Spaces based on the tier.

A background worker built on a shared queue package continuously polls the outbox tables. Employing a skip-locked transaction model, it securely claims and processes deferred events in batches without locking contention. Each worker instance scopes its processing to a specific company slug for isolated, scalable background execution.

Shared TypeScript packages encapsulate core cross-cutting concerns, such as domain logic for project costing and margin analysis, tier-aware configuration, and structured logging. The entire system is instrumented with Sentry for end-to-end trace propagation across the browser, API, and workers. In addition, the API exposes Prometheus metrics, surfacing request latency, database pool state, and queue depth for real-time operational observability.
