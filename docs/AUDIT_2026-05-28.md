# Sitelayer System Audit — 2026-05-28

Full audit across code, deployment/CI, GitHub, DigitalOcean, performance, and security.
Conducted on branch `agent/claude-work/v2-backend-review` (off the v2 UI-overhaul HEAD).

**Bottom line:** the codebase is unusually disciplined (forced Postgres RLS, fully
parameterized SQL, constant-time webhook HMACs, SHA-pinned Actions, immutable
checksummed migrations, working restore drill). The real risks are **operational**:
the production DB is under-provisioned relative to the app's connection demand, and
a few security knobs are loose. Governance findings (branch protection, required
reviewers) are intentionally **de-prioritised** — this is a solo-operated repo.

## 📋 Infra checklist — needs prod/account access (only you can do these)

Ordered by impact. The first is the single most important production fix.

### 1. 🔴 Add a DigitalOcean connection pool + lower app pool (root cause of likely outage)
The app can demand ~40+ connections from a Postgres that allows ~22. Even with the
code default now at 16+4, add a real pooler so the DB cap is never the ceiling:
```bash
# transaction-mode PgBouncer in front of sitelayer_prod
doctl databases pool create 9948c96b-b6b6-45ad-adf7-d20e4c206c66 \
  --name sitelayer-prod-pool --mode transaction --size 18 \
  --db sitelayer_prod --user sitelayer_prod_app
# then point prod DATABASE_URL at the pooler port (25061) and keep
# PG_POOL_MAX (api) + WORKER_PG_POOL_MAX (worker) summed under the pool size.
```
Add a DB **connection-count** alert (current alert policies cover CPU/mem/disk but not connections).

### 2. 🟠 Resize the production database (undersized + no HA)
`db-s-1vcpu-1gb`, single node = no standby, ~22-conn cap, 1 GB working set.
```bash
doctl databases resize 9948c96b-b6b6-45ad-adf7-d20e4c206c66 \
  --size db-s-2vcpu-4gb --num-nodes 1     # ~97 conns, +$15/mo; do in the maint window
```
Consider a standby node before broadening past the pilot.

### 3. 🟠 Patch dependency vulns (3 high, runtime path)
`js-cookie ≤3.0.5` (via `@clerk/backend`), `fast-uri ≤3.1.1` (path traversal),
`brace-expansion` (ReDoS). Run when the active dev session is idle (it shares
`node_modules`):
```bash
cd /home/taylorsando/projects/sitelayer && npm audit fix
# bump @clerk/backend / @clerk/shared to a release on js-cookie >3.0.5
```

### 4. 🟠 Release the orphaned reserved IP ($5/mo, dangling)
```bash
doctl compute reserved-ip list      # confirm 159.203.51.235 is still droplet:null
doctl compute reserved-ip delete 159.203.51.235
```

### 5. 🟡 Confirm security-relevant prod secrets are actually set (not silently empty)
Green deploys do **not** prove these are non-empty. Verify in the GitHub `production`
environment: `INTERNAL_AUTH_TOKEN`, `CLERK_WEBHOOK_SECRET`, `QBO_WEBHOOK_VERIFIER`,
`MESH_WORK_REQUEST_DISPATCH_TOKEN`. An empty webhook verifier ⇒ unauthenticated endpoint.

### 6. 🟡 Confirm DB connection uses the scoped app user over the private VPC host
`DATABASE_URL` (prod) should use `sitelayer_prod_app` over the **private**
`private-sitelayer-db-...` host, never `doadmin`/public. (Trusted-sources already
gate the DB to the two droplets — good. doctl surfaced live `doadmin` creds; rotate
if they were ever embedded in app config.)

### 7. 🟡 Enable Dependabot **security alerts** (separate toggle from the version-update config that's already on) and confirm secret-scanning + push protection (Settings → Code security).

### 8. 🟡 Confirm Spaces is not public-read (PII)
Anonymous bucket *listing* is correctly blocked (403). Per-object ACL/CORS/versioning
couldn't be checked from the agent host — run `docs/RUNBOOK_SPACES_CORS.md` Step 1 with
the account-owner Spaces key to confirm no public-read ACLs and that versioning is on.

## 🔭 Code follow-ups worth a future branch (not yet done)

- **JWT `aud`/`azp` + mandatory-issuer-in-prod.** `auth.ts:verifyClerkJwt` checks
  signature/exp/nbf and issuer *only if* `CLERK_ISSUER` is set; there's no audience/
  authorized-party check. Needs an `AuthConfig` field + env (`CLERK_AUTHORIZED_PARTIES`)
  and a fail-closed-in-prod branch. ~30 min.
- **Rate-limit the portal + auth-failure surface.** `/api/portal/*` is fully exempt and
  limiting runs after identity resolution, so 401-spam isn't throttled. Add a dedicated
  per-IP limiter inline. `rate-limit.ts:105-121`.
- **`/api/analytics` loads full `labor_entries` history per call** (no date filter/LIMIT;
  no index can rescue it). Push aggregation into SQL or default to a trailing window.
  `analytics.ts:48`.
- **Auto-rollback on failed post-deploy health check.** Containers swap + migrations
  apply *before* the health loop; on failure prod is left broken with only manual
  recovery. `deploy-droplet.yml:545-562`.
- **Registry keeps only 2 SHA tags** (`REGISTRY_TAG_KEEP`) — two quick deploys delete the
  rollback target. Bump it or pin last-known-good separately.
- **Worker healthcheck** only proves a node process is PID 1, not that the queue drains —
  a hung worker reports healthy and silently stops QBO sync. `docker-compose.prod.yml:164`.
- **`MIGRATION_GUARD_OVERRIDE: '1'` is hardcoded on every CI run** (`quality.yml:58`) —
  the forward-only-migration immutability guard is effectively disabled. Was a "one-time
  override for 087"; scope it to that branch or remove once 087 is settled.
- **Node version drift:** CI tests on Node 22, prod builds/runs on Node 20; base images
  are floating tags (not digest-pinned) → non-reproducible builds.

## 🟢 Verified good (not problems)

- Multi-tenant isolation: app-level `company_id` filtering **+** forced Postgres RLS on
  all tables (`SET LOCAL app.company_id`); no IDOR found.
- SQL fully parameterized; the only `${}` interpolations are constant column-lists / literal table names.
- Webhook HMACs (QBO `intuit-signature`, Clerk Svix) verified on the **raw** body with constant-time compare, pre-auth.
- The `x-sitelayer-act-as` dev bypass is airtight (two independent prod gates); `AUTH_ALLOW_HEADER_FALLBACK` cannot activate in prod.
- CORS is allowlisted (no wildcard reflection with credentials).
- Blueprint downloads are company-scoped with path-traversal guards; presigned TTL 15 min.
- Firewalls expose only 80/443/22 (3000/3001 private); SSH source-restricted; droplets + DB share a private VPC.
- Backups + off-host copy + restore drill + monitoring alerts are real and live; uptime checks on `/health`.
- Queue uses `FOR UPDATE SKIP LOCKED` correctly with partial ready-indexes; no N+1 in hot paths; index coverage strong.
- Frontend bundle well-split (lazy `vendor-three`, sourcemaps gated, manual vendor chunks).
